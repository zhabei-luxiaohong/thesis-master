# 交叉验证数据表

## 验证原则
1. **所有数据必须来自官方财报**（巨潮资讯网、公司官网、交易所网站）
2. **关键数据点需要两个以上来源交叉验证**（财报+权威媒体报道+专业数据库）
3. **建立完整的数据溯源链**（数据→来源→验证状态→引用格式）
4. **记录不一致数据点并核实原因**

### 6.2 验证流程
1. **数据提取**：从官方年报提取关键数据
2. **数据库核对**：与专业数据库进行对比
3. **差异分析**：分析数据差异原因（会计准则差异、统计口径等）
4. **数据确定**：确定最终使用数据及解释说明

## 7. 数据不一致处理原则

### 7.1 不一致类型
1. **财报与媒体报道不一致**：优先采用财报数据，注明媒体报道作为参考
2. **不同年份财报统计口径变化**：注明调整事项，进行可比性调整
3. **临时会计政策变更影响**：说明影响程度，必要时进行数据调整
4. **市场传闻与官方数据不一致**：以官方数据为准，注明传闻作为市场反应参考

### 7.2 处理流程
```
发现不一致 → 确认来源权威性 → 分析差异原因 → 决定处理方式 → 记录处理说明
```

## 8. 引用格式规范

### 8.1 基本格式
- **财报数据**：`[公司名称+年份+年报][页码/表格名称]`
- **媒体报道**：`[媒体名称+报道时间+文章标题]`
- **专业数据库**：`[数据库名称+数据更新时间+数据代码]`
- **官方文件**：`[发文机构+文件编号+发布日期]`

### 8.2 具体示例
1. **大悦城财报**：`[大悦城控股2024年年度报告][合并利润表]`
2. **华夏REIT招募书**：`[华夏大悦城购物中心REIT招募说明书][2024年现金流分派率预测]`
3. **权威媒体报道**：`[财新网2025年4月15日《大悦城首次年度亏损近30亿元》]`
4. **数据库数据**：`[Wind数据库2026年4月14日大悦城财务数据]`
