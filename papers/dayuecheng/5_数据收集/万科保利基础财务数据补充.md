## 四、DID模型控制组适用性分析

### 4.1 控制组选择合理性

#### 万科作为控制组的合理性：
1. **行业可比性**：同属房地产行业，受相同行业环境影响
2. **规模差异**：规模远大于大悦城，可提供"反事实"比较基准
3. **REITs经验**：有REITs发行经验，但发行目的不同（非困境化解）
4. **数据可得性**：财务数据相对完整，便于实证分析

#### 保利作为控制组的合理性：
1. **央企背景**：与大悦城同为央企控股，治理结构相似
2. **业务相似性**：商业地产+住宅开发业务结构相似
3. **REITs探索**：有REITs早期尝试，但未正式发行
4. **数据挑战**：财务数据相对缺乏，需要较多补充

### 4.2 控制组数据完整性要求

#### DID模型数据要求：
```
1. 时间跨度一致：所有企业2019-2024年完整数据
2. 核心变量完整：营业收入、净利润、总资产、资产负债率
3. 控制变量一致：公司规模、成长性、盈利能力等
```

#### 当前数据缺口与优先级：
| 企业 | 数据完整性 | 关键缺口 | 补充优先级 |
|------|------------|----------|------------|
| **大悦城** | 85% | 总资产数据、2024年完整数据 | 第一优先级 |
| **万科** | 65% | 2022年详细数据、2024年预计数据 | 第二优先级 |
| **保利** | 35% | 几乎所有年份详细数据 | 第三优先级 |

## 六、数据应用与研究设计

### 6.1 DID模型数据流设计

```
数据收集 → 数据清洗 → 变量计算 → 面板构建 → 平行趋势检验 → DID估计 → 稳健性检验
```

#### 关键节点：
1. **数据收集节点**：确保所有企业时间序列完整
2. **变量计算节点**：统一财务比率计算口径
3. **平行趋势检验节点**：严格检验DID模型适用性

### 6.2 实证分析数据准备

#### 需要准备的数据集：
1. **原始数据表**：各企业官方财报提取数据
2. **计算数据表**：财务比率和控制变量计算
3. **面板数据表**：DID模型专用数据结构
4. **检验数据表**：稳健性检验备用数据集

### 6.3 质量保证措施

#### 数据验证措施：
1. **来源双重验证**：至少两个独立数据源验证
2. **逻辑一致性检验**：财务数据逻辑关系验证
3. **异常值识别处理**：识别和处理异常数据点
4. **敏感性分析**：测试不同假设下的结果稳健性
