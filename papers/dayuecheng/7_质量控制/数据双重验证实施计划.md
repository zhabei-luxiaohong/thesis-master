# 数据双重验证实施计划

## 一、验证目标与原则

### 1.1 验证目标
- **100%关键数据**：营业收入、净利润、总资产、资产负债率等核心财务指标
- **100%时间序列**：2019-2024年所有年份数据
- **100%企业覆盖**：大悦城（处理组）+ 万科、保利（控制组）
- **双重验证率**：所有数据点至少两个独立来源验证

### 1.2 验证原则
1. **来源独立性原则**：验证来源必须相互独立
2. **官方优先原则**：官方财报数据为基准
3. **及时更新原则**：数据发现不一致立即核实更新
4. **完整记录原则**：所有验证过程完整记录

## 三、验证流程设计

### 3.1 单一数据点验证流程
```
第1步：数据发现 → 第2步：官方财报验证 → 第3步：权威媒体验证 → 第4步：数据库验证 → 第5步：结果记录
```

#### **验证标准**：
- **合格标准**：至少一个一级来源+一个二级/三级来源验证一致
- **可疑标准**：不同来源数据不一致
- **不合格标准**：单一来源，无法验证

### 3.2 批量数据验证流程
```
第1步：建立待验证清单 → 第2步：来源优先级排序 → 第3步：并行验证实施 → 第4步：不一致处理 → 第5步：质量报告生成
```

### 5.2 验证脚本设计

#### **数据一致性检查脚本**：
```python
def verify_data_consistency(data_point, sources):
    """
    验证数据点一致性
    data_point: 待验证数据点
    sources: 验证来源列表
    返回：验证结果（合格/可疑/不合格）
    """
    # 检查来源数量
    if len(sources) < 2:
        return "不合格"
    
    # 检查数据一致性
    values = [source['value'] for source in sources]
    if all(v == values[0] for v in values):
        return "合格"
    elif max(values) - min(values) / values[0] < 0.05:  # 5%以内差异
        return "合格（轻微差异）"
    else:
        return "可疑"
```

#### **批量验证脚本**：
```python
def batch_verification(data_table, verification_sources):
    """
    批量验证数据表
    data_table: 待验证数据表
    verification_sources: 验证来源配置
    返回：验证报告
    """
    results = []
    for data_point in data_table:
        verification_result = verify_data_point(data_point, verification_sources)
        results.append(verification_result)
    
    return generate_verification_report(results)
```

## 七、不一致数据处理

### 7.1 不一致类型分类

#### **轻微不一致**（差异<2%）：
- 处理方式：取官方财报数据为准
- 记录方式：标记轻微差异，记录所有值

#### **中度不一致**（差异2-5%）：
- 处理方式：多方来源核实，取多数一致值
- 记录方式：详细记录差异分析过程

#### **严重不一致**（差异>5%）：
- 处理方式：暂停使用，深度调查原因
- 记录方式：建立专门调查记录

### 7.2 不一致处理流程
```
发现不一致 → 来源优先级排序 → 多方来源核实 → 专家咨询（必要时） → 确定最终值 → 更新所有文件 → 记录处理过程
```
