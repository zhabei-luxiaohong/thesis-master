## 🤖 AI执行标准（内嵌，用户无需填写）

### 一、理论框架（5选+AI推荐）

| 编号 | 框架名称 | 适用场景 | 核心要素 |
|------|----------|----------|----------|
| F1 | Yin案例研究框架 | 单案例深度/多案例比较 | 单案例→多案例→跨案例分析 |
| F2 | 事件研究法 | 政策/事件冲击效应 | 事件窗口→累计超额收益→显著性检验 |
| F3 | 财务穿透分析 | 企业财务效应研究 | 五维度：资产轻量化/收入结构/盈利质量/现金流/资本效率 |
| F4 | 6D分析框架 | REITs/商业地产研究 | 筛选/结构/运营/风险/跨境/价值链 |
| F5 | DID双重差分 | 政策效应评估 | 处理组vs对照组→平行趋势检验→稳健性检验 |
| AI | AI智能推荐 | 根据研究问题自动匹配 | 基于文献库+研究问题特征 |

**AI执行规则**：
1. 用户选择后，AI输出该框架的详细操作步骤
2. 若选"AI推荐"，AI基于研究命题特征推荐最优框架并说明理由
3. 每个框架配套：分析维度表+数据需求清单+验证标准

### 二、研究方法与数据验证

#### 2.1 证据三角形验证
```
文档证据（年报/公告/研报）
    +
访谈证据（如有）
    +
观察证据（公开信息/行业报告）
    ↓
交叉验证，单一来源需标注局限性
```

#### 2.2 三阶段验证框架（案例研究必备）
| 阶段 | 时间窗口 | 验证重点 | 核心指标 |
|------|----------|----------|----------|
| 短期 | 0-3月 | 市场反应/事件效应 | 累计超额收益/成交量变化 |
| 中期 | 3-12月 | 经营改善/结构调整 | 营收增速/资产负债率 |
| 中长期 | 12月+ | 战略转型/可持续性 | 经常性收入占比/ROE |

#### 2.3 数据双重验证
- **来源1**：Wind/Choice/同花顺（官方数据库）
- **来源2**：企业年报/公告（原始来源）
- **规则**：关键数据必须双源验证，不一致时以年报为准并说明差异

### 三、论文结构（IMRaD标准）

```
第一章 绪论（漏斗式开篇）
├── 1.1 研究背景（宏观→行业→企业）
├── 1.2 研究意义（理论+实践）
├── 1.3 研究内容与方法
├── 1.4 研究框架与创新点
└── 1.5 论文结构

第二章 文献综述与理论基础
├── 2.1 核心概念界定
├── 2.2 理论基础（详细阐述所选框架）
├── 2.3 文献综述（Q1-Q4矩阵）
└── 2.4 文献评述与研究空白

第三章 研究设计
├── 3.1 研究问题细化
├── 3.2 案例选择依据（代表性/极端性/独特性）
├── 3.3 数据来源与采集方法
├── 3.4 分析框架构建
└── 3.5 研究方法详述

第四章 案例分析（核心章节）
├── 4.1 案例企业简介
├── 4.2 核心矛盾提炼（叙事驱动开篇）
├── 4.3 分析维度展开（基于所选框架）
├── 4.4 机制验证（数据→机制→结论）
└── 4.5 稳健性检验

第五章 研究结论与建议
├── 5.1 主要发现（回答研究问题）
├── 5.2 理论贡献（方法论/机制发现）
├── 5.3 实践建议（企业/监管）
├── 5.4 研究局限
└── 5.5 未来研究方向
```

### 四、写作规范（自动执行）

#### 4.1 时态规范
| 章节 | 时态 | 示例 |
|------|------|------|
| 引言/文献综述 | 现在时 | "REITs是一种..." |
| 研究方法 | 过去时 | "本研究采用了..." |
| 研究结果 | 过去时 | "数据显示..." |
| 讨论/结论 | 现在+过去时 | "这一发现表明...本研究发现..." |

#### 4.2 模糊限制语规范
| 禁用词 | 替换词 |
|--------|--------|
| 证明 | 表明/显示/提示 |
| 导致 | 可能相关/有关联 |
| 必然 | 往往/通常 |
| 确定 | 提示/暗示 |

#### 4.3 引用格式
- **MBA论文标准**：GB/T 7714-2015
- **格式示例**：[1] 张三. REITs研究[J]. 金融研究, 2024(3): 45-56.
- **工具要求**：Zotero/EndNote自动生成，统一格式

#### 4.4 图表规范
- 表格标题在上，图表标题在下
- 连续编号（表1、表2.../图1、图2...）
- 文中必须引用（"如表1所示"、"见图2"）
- 数据来源标注（数据来源：Wind，作者整理）

### 五、质量标准（5大阶段里程碑）

#### 阶段一：选题与设计（2-3周）
```
里程碑1：选题确认
├── 输出：选题评估矩阵（3-5个候选）
│   └── 评分维度：创新性(5)/可行性(5)/数据可得性(5)/理论贡献(5)，总分≥16分通过
├── 输出：导师沟通记录
└── 决策点：⏸️ 用户确认最终选题

里程碑2：研究设计
├── 输出：开题报告初稿（赖一飞9部分结构）
└── 决策点：⏸️ 用户确认研究框架
```

#### 阶段二：文献与数据（3-4周）
```
里程碑3：文献矩阵
├── 输出：Q1-Q4文献矩阵（50篇核心文献）
│   ├── Q1：理论框架+核心概念
│   ├── Q2：研究对象特征
│   ├── Q3：研究方法应用
│   └── Q4：研究结论对比
└── 决策点：⏸️ 用户确认理论框架

里程碑4：数据收集
├── 输出：数据清单+数据来源说明
├── 输出：数据完整性检查报告
└── 决策点：⏸️ 用户确认数据完整性
```

#### 阶段三：分析与撰写（4-6周）
```
里程碑5：章节撰写
├── 输出：逐章初稿（按IMRaD结构）
├── 质量：每章提交前AI自检（P1/P2/P3问题清单）
└── 决策点：⏸️ 用户确认每章质量

里程碑6：逻辑整合
├── 输出：全文初稿
├── 输出：逻辑检查报告（章节衔接/论证链完整性）
└── 决策点：⏸️ 用户确认整体逻辑
```

#### 阶段四：修改与完善（2-3周）
```
里程碑7：质量检查
├── 输出：质量检查清单
│   ├── P1级（必须修复）：研究问题不明确/方法论缺陷/核心论证断裂
│   ├── P2级（建议修复）：文献综述不系统/分析不深入/语言不学术
│   └── P3级（可选修复）：格式细节/引用不统一
└── 决策点：⏸️ 用户确认修改优先级

里程碑8：语言优化
├── 输出：去AI化报告（AI痕迹检测10大特征）
├── 输出：语言自然化版本
└── 决策点：⏸️ 用户确认最终版本
```

#### 阶段五：答辩与提交（1-2周）
```
里程碑9：答辩准备
├── 输出：PPT（15-20页，每页一个核心观点）
├── 输出：答辩陈述稿（8-10分钟）
├── 输出：Q&A预案（10-15个常见问题）
└── 决策点：⏸️ 用户确认答辩材料
```

### 六、团队协作（6个Agent自动分工）

| Agent | 职责 | 核心技能 | 输出物 |
|-------|------|----------|--------|
| 📚 市场调研员 | 文献检索、综述撰写 | Q1-Q4矩阵、布尔检索 | 文献矩阵.xlsx、综述初稿 |
| 📊 数据分析师 | 数据采集、统计分析 | 6D框架、证据三角形 | 数据集、分析报告 |
| ✍️ 写作编辑 | 章节撰写、逻辑整合 | IMRaD结构、叙事驱动 | 章节草稿、全文初稿 |
| 📋 格式规范员 | 引用格式、排版检查 | GB/T 7714、图表规范 | 格式检查清单、排版终稿 |
| 🔬 模式验证员 | 方法论一致性检查 | Yin框架、机制验证 | 方法论检查报告 |
| 🎨 语言优化员 | 去AI化、学术语言 | Hemingway Pass、模糊限制语 | 去AI化报告、终稿 |

**协作机制**：
- 每阶段结束，各Agent输出工作成果
- 项目经理（AI主进程）整合后汇报用户
- 决策点暂停，等待用户确认后进入下一阶段

### 八、踩坑预警（自动检测）

| 陷阱类型 | 症状 | AI自动检测与提示 |
|----------|------|------------------|
| 方法论冲突 | DID+案例研究混用 | ⚠️ 检测到方法论冲突，建议明确主方法 |
| 数据口径混淆 | A股+港股数据混用 | ⚠️ 检测到跨市场数据，请统一口径 |
| 核心矛盾遗漏 | 案例分析无矛盾铺垫 | ⚠️ 建议开篇提炼核心矛盾 |
| 证据链断裂 | 数据与结论无逻辑链接 | ⚠️ 数据→机制→结论三段论不完整 |
| AI痕迹明显 | "首先...其次...最后..." | ⚠️ 检测到AI痕迹，启动去AI化流程 |

```
步骤1：数据搜索

└── 输出：初步数据清单+文献库

步骤2：选题验证
├── AI：输出选题评估矩阵（3-5个候选）
└── ⏸️ 决策点：用户确认选题

步骤3：开题报告
├── AI：输出开题报告初稿
└── ⏸️ 决策点：用户确认研究框架

步骤4：文献矩阵
├── AI：输出Q1-Q4文献矩阵
└── ⏸️ 决策点：用户确认理论框架

步骤5：逐章撰写
├── AI：按IMRaD结构逐章输出
├── 每章提交前：AI自检质量

步骤6：质量检查

步骤7：答辩准备
├── AI：输出PPT+答辩陈述稿+Q&A预案
```

## 🔧 AI执行规则（内置）

1. **触发即执行**：用户填写完成后，AI直接调用工作流，不等待逐步指示
2. **决策点暂停**：每个里程碑后暂停，等待用户确认
3. **质量内嵌**：AI在输出前自检质量，不输出P1级问题内容
4. **踩坑预警**：自动检测常见陷阱并提示
5. **团队协作**：6个Agent按阶段分工，项目经理整合汇报
